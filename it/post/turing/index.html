Macchina di Turing: automa

2019-11-27 — Scritto da Marco
#Turing  #Automa  #MicroBit  #Sequenza 

Introduzione

TuringAutomaton è un progetto open source disponibile sotto licenza GNU General Public License v3.0.

Questo progetto si basa sugli automi, e per cominciare, è meglio dire qualcosa a riguardo.

In teoria dei sistemi dinamici, un automa è un sistema dinamico discreto (nella scansione del tempo e nella descrizione del suo stato) e tempo-invariante (il sistema si comporta alla stessa maniera indipendentemente dall'istante di tempo in cui agisce).

È quindi un apparato che si evolve nel tempo, cambiando la sua condizione in relazione a quella attuale. La sua evoluzione parte da una particolare situazione detta “stato iniziale”.

Quando l’automa si trova in un certo stato, esso lo può cambiare solo ricevendo un sottoinsieme dei simboli del suo alfabeto, cioè quell'insieme che contiene gli elementi che esso può accettare.

In genere gli automi sono deterministici, proprio come TuringAutomaton, ovvero dato uno stato ed un simbolo in ingresso è possibile una sola transizione. Esistono comunque i non deterministici o stocastici.

Una macchina a stati finiti è un tipo di automa che permette di descrivere con precisione e, in maniera formale, il comportamento di molti sistemi.
Grazie alla sua semplicità e chiarezza, questo modello è molto diffuso nell'ingegneria e nelle scienze, soprattutto nel campo dell'informatica e della ricerca operativa.

Un automa a stati finiti può essere utilizzato sia per modellare un sistema esistente, che per modellare un nuovo sistema formale in grado di risolvere alcuni problemi esistenti.
La sua rappresentazione grafica è detta grafo.

Una loro applicazione tipica è il riconoscimento di sequenze di simboli come avviene, per esempio, nei compilatori e negli interpreti dei linguaggi di programmazione per riconoscere la correttezza sintattica delle istruzioni.

TuringAutomaton è proprio un riconoscitore di sequenza che prende forma dalle basi della Macchina di Turing per svolgere le sue operazioni. Analizza infatti carattere per carattere costruendo una stringa che confronta con quella impostata per trovare la sequenza.

Quest'ultimo è disponibile in C, Java, Python, Dart, MicroBit (Javascript), Calc e in UML.
La documentazione è adoperabile sia in Italiano che in Inglese.

Tutti i programmi sono stati ottimizzati e preparati per rispecchiare il funzionamento dell’automa.

È possibile contribuire e trovare i file del progetto su Github.

Funzionamento

L’automa ha bisogno di alcune informazioni primarie per svolgere il suo compito, richiede infatti 5 insiemi e delle tabelle che definiscono il suo comportamento.
È stato definito come alfabeto da gestire, l'insieme  A B C  per la formazione delle sequenze è stata presa in considerazione  AACBB  come sequenza da riconoscere.

Insiemi

Sono stati inizialmente definiti i 5 insiemi che sono rispettivamente: insieme Input, insieme Valori Input, insieme Output, insieme Valori Output e insieme Stati.

Il primo insieme definisce il tipo di ingresso accettato dall’automa che in questo caso è lettera perché “A”, “B” e “C” sono tutte lettere.

L'insieme Valori Input contiene i valori che l'ingresso può assumere cioè, come già detto, i valori “A”, “B” e “C”.

L'insieme Output invece indica il tipo di output che il sistema invierà in uscita.

L'insieme Valori Output rappresenta invece i valori che possono fluire in Output.

L'ultimo insieme cioè quello degli Stati contiene i vari “momenti” che cambiano in base all'ingresso ricevuto e che l'apparato deve attraversare per identificare la sequenza.

  • I = { lettera }
  • VI = { A, B, C }
  • O = { stringa }
  • VO = { AACBB, no }
  • S = { attesa, a, aa, aac, aacb }

Tabella Transizione degli Stati

VI / S attesa a aa aac aacb
A a aa aa a a
B attesa attesa attesa aacb attesa
C attesa attesa aac attesa attesa

Questa tabella descrive come l’automa cambi il suo stato in base all'ingresso ricevuto.

Per esempio, se si trova nello stato aac e riceve in ingresso una c, esso ritorna nello stato iniziale perché quell'ingresso non fa parte della stringa da rilevare.

Tabella Trasformazione delle Uscite

VI / S attesa a aa aac aacb
A no no no no no
B no no no no aacbb
C no no no no no

In quest'altro schema troviamo invece la trasformazione delle uscite, ovvero il valore che il sistema manderà in uscita in seguito allo stato in cui si trova e all'ingresso ricevuto.

Per esempio, se si trova nello stato aac e riceve in ingresso una c, esso manderà in output la stringa no perché l'ingresso inserito non permette l'individuazione della stringa.

UML

Per descrivere al meglio l’automa è stato creato un modello UML, nel quale vengono riportati le 5 situazioni (nodi) e le relative transizioni, segnate dall'ingresso e dalle uscite secondo le tabelle costruite sopra.

“ATTESA”, in questo caso, è indicato come stato iniziale.

Modello Calc

Il tutto è stato messo insieme per creare un modello in calc in modo da poterne testare le funzionalità.

Sono stati creati due fogli di calcolo; Uno per la descrizione nel quale sono state implementate le varie tabelle e il grafo in UML. L'altro invece mostra l'evoluzione del sistema nel tempo in seguito all'ingresso ricevuto.

Attraverso funzioni speciali (CERCA.ORIZZ, CERCA.VERT e INDIRETTO) sono stati presi i valori nelle tabelle della descrizione per la formazione della tabella con l'evoluzione dell’automa.

Implementazione Microbit

Lo stesso programma è stato implementato su un dispositivo fisico: MicroBit.
Una schedina programmabile, semplice e intuitiva da utilizzare su cui è stata testata l’automa.

Il codice è stato ottimizzato per rendere riconoscibile qualsiasi sequenza con un massimo di tre caratteri per la formazione della stringa (Limitati dalle funzionalità di microbit. In futuro sarà magari possibile estendere questo limite).

Funzionamento

Il funzionamento del programma è piuttosto semplice:

  • Nel tasto A è associata la prima lettera in questo caso “A”
  • Nel tasto B è associata la seconda lettera in questo caso “B”
  • Con il movimento è associata la terza lettere in questo caso “C”

Il tipo di output può essere a “numero” (come nell'immagine che seguirà) cioè viene identificato il numero di volte che la sequenza viene riconosciuta all'interno dell'array.

oppure può essere a “stringa” (come nell'immagine che seguirà) ovvero viene mostrato a schermo l'intero array, più precisamente, l'intera stringa e si mostra con l'avanzamento dei caratteri una faccina sorridente a stringa identificata.

Qui sotto è presente il codice sorgente del meccanismo usato per far funzionare microbit.

L'array rappresenta lo stato dell’automa infatti ogni volta che viene inserito un valore cambia secondo il suo valore precedente.

// listen for the buttons A

    pushArray(alphabet[0])
})
// B
input.onButtonPressed(Button.B, function () {
    pushArray(alphabet[1])
})
function checkArray() {
    // Check array function
    //
    // check if the inserted string has the string
    //
    // Parameters:
    //
    // Return:
    if (state.join("").includes(STRING)) {
        stringCount += 1
        basic.showIcon(IconNames.Happy)
    }
    if (outputType == "string") {
        basic.clearScreen()
        basic.showString("" + state.join(""))

let stringCount = 0
let outputType = ""
let state: string[] = []
let alphabet: string[] = ["A", "B", "C"]
let STRING = "AACBB"
function pushArray(value = "") {
    state.removeAt(0);
    state.push(value);
    checkArray()
}
outputType = "stringCount" // string or stringCount
// listen the shake for the c value
input.onShake(function () {
    pushArray(alphabet[2])
})
for (let index = 0; index < STRING.length; index++) {
    state.push("")
}
// show counter if the output type is set to
// stringCount
basic.forever(function () {
    if (outputType == "stringCount") {
        basic.showNumber(stringCount)
    }
})

Implementazione in C

Funzionamento

Il programma ti chiederà di volta in volta di inserire un carattere, il quale verrà inserito in un array che sarà esaminato per trovare la stringa richiesta. Se l'esito è positivo verrà stampata la stringa “Sequence found!".

L'array nel quale verranno salvati tutti i caratteri sarà della stessa dimensione della stringa da trovare di conseguenza piu la stringa sarà lunga e più memoria verrà occupata.

Codice e spiegazione

Qui è riportato il codice sorgente.

Nella variabile costante STRING è riportata la stringa da controllare nella sequenza di input.

Nella variabile STRING_LENGTH è definita la lunghezza della stringa (Attenzione questa costante deve sempre essere definita e deve corrispondere alla lunghezza della stringa).

Nel metodo main() vengono richiesti di volta in volta i parametri e inseriti all'interno dell'array con la relativa funzione pushArray(char) la quale sposta di 1 l'array a sinistra per inserire il nuovo carattere.

La funzione resetArray() pulisce l'array e può anche essere utilizzata per inizializzarlo.

La funzione checkArray() controlla la stringa all'interno dell'array e ne stampa l'esito.

/**
    * AUTHOR: Condrache Marco Mihai
    * CLASS: 3^A 
    * DATE: 28/11/2019
    * DESCRIPTION: Program for the __automa__ton based on the concept of turing machine
    * ? GITHUB: https: //github.com/MarcoMihaiCondrache
*/

#include <stdio.h>
#include <string.h>
#define STRING "aacbb" // STRING that will be searched into the array
#define STRING_LENGTH 5

// Declaring variables
char word[STRING_LENGTH];
void resetArray();
void pushArray(char);
void checkArray();

int main(int argc, char const *argv[])
{
    char current = STRING[0];
    resetArray();
    // ask for the chars
    while (strchr(STRING, current) != NULL)
    {
        printf("Insert char: ");
        scanf("%1s", &current);
        pushArray(current);
        checkArray();
    }
}

/**
 * Method that reset array to * values
 * @return void
 */
void resetArray()
{
    for (int i = 0; i < STRING_LENGTH; i++)
    {
        word[i] = '*';
    }
}
/**
 * Method that shift by 1 the array and append the value
 * @param char The value to append
 * @return void
 */
void pushArray(char value)
{
    for (int i = 0; i < STRING_LENGTH - 1; i++)
    {
        word[i] = word[i + 1];
    }
    word[STRING_LENGTH - 1] = value;
}
/**
 * Check if the array containts the STRING value
 * @return void
 */
void checkArray()
{
    if (strncmp(word, STRING, strlen(STRING)) == 0)
    {
        printf("Sequence found!\n");
    }
}

Implementazione in Java

Funzionamento

Il programma ti chiederà di volta in volta di inserire un carattere, il quale verrà inserito in un array che verrà a sua volta esaminato per trovare la stringa richiesta. Se l'esito è positivo verrà stampata la stringa “Sequence found!”.

Codice e spiegazione

Di seguito è riportato il codice sorgente.

La classe TuringAutomaton è la classe dove sono definiti i metodi per il funzionamento dell’automa Il metodo main contiene l'inzio del programma quindi deve essere richiamato per inizializzare il programma. È di tipo statico e pubblico.

Nella variabile costante STRING_TO_FIND è riportata la stringa da controllare nella sequenza di input.

Il metodo pushInArray viene richiamato per inserire il carattere all'interno dell'array secondo il concetto LIFO.

Il metodo checkArray viene richiamato per controllare nell'array la stringa e ne restituisce l'esito.

import java.util.ArrayList;
import java.util.Scanner;

/**
 * Program based on the Turing Machine concept (console)
 * <br>
 * https://github.com/MarcoMihaiCondrache/TuringAutomaton
 *
 * @author  Condrache Marco Mihai
 * @version 1.0
 * @since   2019-11-28
 */

public class TuringAutomaton {
    /**
     * The final string that contains the string to find
     */
    private static final String STRING_TO_FIND = "aacbb";

    public static void main(String[] args) {
        String current;
        // full word to compare
        ArrayList<String> word = new ArrayList<String>();
        for (int i=0; i<STRING_TO_FIND.length(); i++){
            word.add("");
        }
        do {
            // insert and check the word with the new char
            Scanner reader = new Scanner(System.in);
            System.out.print("Insert char: ");
            current = reader.nextLine();
            word = pushInArray(word, current);
            if (checkWord(word)){
                System.out.println("Sequence found!\n");
            }
        }while (STRING_TO_FIND.contains(current));
        System.out.println(current);
    }

    /**
     * Method to check if the list contains
     * the STRING_TO_FIND and return true or false.
     *
     * @param  list The array to join to compare
     * @return  if the string contains the cost
     * @since 1.0
     */
    private static boolean checkWord(ArrayList<String> list){
        StringBuilder sb = new StringBuilder();
        for (String s : list)
        {
            sb.append(s);
        }
        return sb.toString().contains(STRING_TO_FIND);
    }

    /***
     * Method to push into array using LIFO concept
     *
     * @param list The array to push in
     * @param charT The char to insert
     * @return return the complete list
     */
    private static ArrayList<String> pushInArray(ArrayList<String> list, String charT){
        list.remove(0);
        list.add(charT);
        return list;
    }
}

Implementazione in Python

Funzionamento

Il programma ti chiederà di volta in volta di inserire un carattere, il quale verrà inserito in un array che verrà a sua volta esaminato per trovare la stringa richiesta. Se l'esito sarà positivo verra stampata la stringa “Sequence found”.

Codice e spiegazione

Di seguito è riportato il codice sorgente.

Nella variabile costante STRING è riportata la stringa da controllare nella sequenza di input.

Vengono richiesti di volta in volta i parametri e inseriti all'interno dell'array con la relativa funzione push_array(char) la quale sposta di 1 l'array a sinistra.

La funzione check_array() controlla la stringa all'interno dell'array e ne restituisce l'esito.

"""
    * AUTHOR: Condrache Marco Mihai
    * CLASS: 3^A
    * DATE: 28/11/2019
    * DESCRIPTION: Program based on the Turing Machine concept (console)
    ? GITHUB: https://github.com/MarcoMihaiCondrache/TuringAutomaton
"""

# Define STRING variable to find inside characters
STRING = "aacbb"
# Define word variable that will contain the insert string
word = [""] * len(STRING)
# Define current variable that will contain the current char
current = STRING[0]


def push_array(char):
    """
    Push into array with Lifo concept

    :param char: The char to insert into the array
    :return:
    """
    word.pop(0)
    word.append(char)


def check_array():
    """
    Check if the array contains the sequence

    :return bool: Return the condition
    """
    return STRING in "".join(word)


while current in STRING:
    # Ask the char
    current = str(input("Insert char: "))
    # Insert it into the array
    push_array(current)
    # Check and eventually clear the array
    if check_array():
        print("Sequence found")

# If the while statement exits means that it has received an invalid char
print("Invalid char")

Implementazione in Dart

Funzionamento

Il programma ti chiederà di volta in volta di inserire un carattere, il quale verrà inserito in un array che verrà a sua volta esaminato per trovare la stringa richiesta. Se l'esito è positivo verra stampata la stringa “Sequence found”.

Codice e spiegazione

Di seguito è riportato il codice sorgente.

Nella variabile costante STRING è riportata la stringa da controllare nella sequenza di input.

La funzione check_array() controlla la stringa all'interno dell'array e ne restituisce l'esito.

import 'dart:io';

/// Constant string that have the value to find inside the array
/// Type: [String]
const STRING = "aacbb";

/// Main method that request the char every time
/// Return: [void]
void main() {
  /// Generate a list with empty space to prevent [Segmentation Fault]
  List<String> word = List.generate(STRING.length, (index) => "");

  /// First char of the string
  String current = STRING.split("")[0];
  do {
    print("Insert char: ");
    current = stdin.readLineSync();

    /// Save chars with the LIFO concept
    word.removeAt(0);
    word.add(current);

    /// Check the string
    if (checkArray(word)) {
      print("Sequence found!");
    }
  } while (STRING.contains(current));
}

/// Method that check if the array contains the [STRING]
/// Param: [List<String> array]
/// Return: [bool]
bool checkArray(List<String> array) {
  return array.join("") == STRING ? true : false;
}
