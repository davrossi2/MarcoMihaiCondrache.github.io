How the cpu is made?

2019-10-01 — Written by Marco
#32bit  #64bit  #how  #cpu 

What is its purpose?

It is a hardware device whose purpose is to execute and process instructions starting from an architecture under the control of the operating system

It is called a “central” unit because it centrally coordinates all the other processing units present in the computer hardware architecture for managing the various internal peripherals or electronic cards (sound card, video card, network card), eg. coprocessor and digital signal processor.

Based on the memory organization, the following two CPU families can be distinguished:

  • with von Neumann architecture, in which data and instructions reside in the same memory (it is therefore possible to have self-modifying code). This architecture is the most common, because it is very simple and flexible.
  • with Harvard architecture, in which data and instructions reside in two separate memories. This architecture can guarantee better performance since the two memories can work in parallel reducing the structural wings, but it is obviously much more complex to manage. It is typically used in DSPs.

Structure

A generic CPU contains:

  • a control unit (also known as CU) which reads the instructions from the memory, if necessary it also reads the data for the instruction read, executes the instruction and memorizes the result if there is, writing it in memory or in a CPU register.

  • an arithmetic and logical unit (also known by the acronym ALU) that deals with performing the logical and arithmetic operations;

  • The registers or small memories used by the CPU to carry out the instructions:        - the register PC (Program Counter), which contains the address in memory of the next instruction to be executed;

  - the register MAR (Memory Address Register) records the address of the address of the memory location being accessed;

  - the register MDR (Memory Data Register) All the data and instructions that must be processed in the processor from the memory, pass through the MDR register and only subsequently from this reach the appropriate registers for the actual processing;

  - the register IR (Instruction Register) This register has the task of receiving from the memory (through the MDR), during a fetch phase, the instruction to be executed, that is the one pointed by the PC;

  - registers general (R1, R2 ..) They are used to contain the data in transit for an elaboration: the addends of an addition that the ALU is about to carry out, the result of a calculation that the ALU has carried out, a memory address where there is a datum that will have to be accessed later, etc. A large number of such registers confers greater flexibility in programming, but complicates the structure of the processor from an architectural point of view.    Usually the size of the registers and buses is equal to the size of the instructions that the cpu must execute. For example, if the cpu executes 64bit instructions, the registers will be 8 bytes (8 * 8 = 64)   

The execution of an instruction

   1. Loading of the instruction (Instruction Fetch) from the memory based on the address provided by the registers.    2. Decoding (Operand Assembly): once the word has been taken, it is determined which operation should be performed and how to obtain the operands, based on a function whose domain consists of the operating codes (typically the high bits of the word and the codomain consists of the microprograms to be executed.    3. Execution (Execute) The instruction is executed: In the last step of the execution the PC is incremented: typically of one if the instruction was not a conditional jump, otherwise the increase depends on the instruction and the outcome of this  

Synchronization

The cpu to execute the instructions needs to be synchronized with the other components of the PC such as with the memory. This task takes care of the clock.

The term clock, In electronics, indicates a periodic signal, generally a square wave, used to synchronize the functioning of digital electronic devices.

The clock speed or frequency is the number of commutations between the two logic levels “0” and “1” that the circuits inside a computing unit or a microprocessor are able to perform in the time unit of a second, and is expressed in cycles per second, or hertz, and its multiples; normally to execute an instruction or a simple sum more clock cycles are needed.
