Turing Machine: Automaton

2019-11-27 — Written by Marco
#Turing  #Automa  #MicroBit  #Sequence 

TuringAutomaton is an open source project available under the GNU General Public License v3.0.

This project is based on automata and to start with it is better to say something about it.

In dynamical systems theory, an automaton is a discrete dynamic system (in the time and description of its state) and time-invariant (the system behaves in the same way regardless of the instant of time in which it acts).

An automaton is therefore a system that evolves over time by changing its state in relation to the current one.

When the automaton is in a given state, it can change its state only by receiving a subset of the symbols of its alphabet that is, that set containing the elements that the automaton can accept.
The evolution of an automaton starts from a particular state called initial state.

Typically, automata are deterministic, just like TuringAutomaton, meaning that only one transition is possible given an input status and symbol. However, there are also non-deterministic or stochastic automata.

A finite-state automaton or finite-state machine is a type of automaton that allows the behavior of many systems to be described accurately and formally.
Thanks to its simplicity and clarity, this model is widespread in engineering and sciences, especially in the field of computer science and operational research.

A typical application of automata is the recognition of symbol sequences, as happens for example in compilers and interpreters of programming languages ​​to recognize the syntactical correctness of the instructions (operation called parsing).

TuringAutomaton is just a automaton sequence recognizer that takes shape from the basics of the Turing Machine to carry out its operations. In fact, it analyzes character by character by constructing a string that compares with the one set to find the sequence.

TuringAutomaton is available in C, Java, Python, Dart, MicroBit (Javascript), Calc and UML.
The documentation can be used both in Italian and in English.

All programs have been optimized and prepared to reflect the automaton function.

You can contribute and find the project files on Github.

How does it work?

The automaton needs some primary information to do its job.
In fact, it requires 5 sets and tables that define its behavior.
The automaton in question has the   A B C   for the formation of sequences and as a sequence from specification  AACBB 

Sets

The 5 sets that are respectively initially defined are: Input set, Input values ​​set, Output set, Output values ​​set and States set.

The first set defines the type of input accepted by the automaton which in this case is lettera because “A”, “B”, and “C” are all letters.

The Input Values ​​set contains the values ​​that the input can assume, that is, as already said, the values ​​"A”, “B” and “C”.

The Output set instead indicates the type of output that the automaton will send to the output.

The Set Values ​​Output represents instead the values ​​that can flow in Ouput

The last set, ie that of the states, contains the various “moments” that change according to the input received and that the automaton must cross to identify the sequence.

  • I = { char }
  • VI = { A, B, C }
  • O = { string }
  • VO = { AACBB, no }
  • S = { wait, a, aa, aac, aacb }

State Transition Table

VI / S wait a aa aac aacb
A a aa aa a a
B wait wait wait aacb wait
C wait wait aac wait wait

This table describes how the automaton changes its state based on the input received.

For example, if the automaton is in the aac state and receives an input c it returns to the initial state because that input is not part of the string to be detected.

Output Transformation Table

VI / S wait a aa aac aacb
A no no no no no
B no no no no aacbb
C no no no no no

In this other table instead we find the transformation of the outputs that is what value the automaton will send in output following the state in which it is found and to the received input.

For example, if the automaton is in the aac state and receives an c input, the automaton will output the string no because the input entered does not allow the string to be identified.

UML

To best describe the automaton a UML model has been created where the 5 states of the automaton (nodes) and the relative transitions marked by the entrance and exits are reported according to the tables built above.

“WAIT” is also referred to as the initial state ie the state from which the automaton will begin to evolve.

Calc Model

Everything has been put together to create a calc model so that you can test its functionality.

Two spreadsheets have been created. One for the description in which the various tables and the UML graph have been implemented. The other sheet instead shows the evolution of the automaton in time following the input received.

Through special functions (SEARCH.HORIZ, SEARCH.VERT and INDIRECT) we have taken the values ​​in the tables of the description for the formation of the table with the evolution of automaton.

Microbit implementation

The same program was implemented on a physical device: MicroBit.
A programmable, simple and intuitive to use card on which the automaton has been tested

The code has been optimized to make any sequence recognizable with a maximum of three characters for the formation of the string (Limited by microbit functionality. In the future it will perhaps be possible to extend this limit).

Operation

The operation of the program is quite simple:

  • In the A key the first letter is associated in this case “A”
  • In the key b the second letter is associated in this case “B”
  • With the movement the third letter is associated in this case “C”

The type of output can be “number” (as in the image that follows) the number of times the sequence is recognized within the array is identified.

Or it can be a “string” (as in the image that follows) the entire array is shown on the screen, that is the entire string and during the progress of chars a smile face will be showed at recognised string

Implementation in C

Operation

The program will ask you from time to time to insert a character, which will be inserted into an array which will in turn be examined to find the string requested. If the result is positive, the string “Sequence found!” Will be printed.

The array where all characters will be saved will be the same size as the string to be found consequently the longer the string will be, the more memory will be occupied.

Code and explanation

Below is the source code

The constant variable STRING contains the string to be checked in the input sequence

In the variable STRING_LENGTH the length of the string is defined (Warning this constant must always be defined and must correspond to the length of the string)

In the main () method, the parameters and pushats are requested in the array with the relative function pushArray(char) which shifts the array to the left by 1

The resetArray() function cleans the array and is used to initialize it

The checkArray() function checks the string inside the array and prints the result

/**
    * AUTHOR: Condrache Marco Mihai
    * CLASS: 3^A 
    * DATE: 28/11/2019
    * DESCRIPTION: Program for the automaton based on the concept of turing machine
    * ? GITHUB: https: //github.com/MarcoMihaiCondrache
*/

#include <stdio.h>
#include <string.h>
#define STRING "aacbb" // STRING that will be searched into the array
#define STRING_LENGTH 5

// Declaring variables
char word[STRING_LENGTH];
void resetArray();
void pushArray(char);
void checkArray();

int main(int argc, char const *argv[])
{
    char current = STRING[0];
    resetArray();
    // ask for the chars
    while (strchr(STRING, current) != NULL)
    {
        printf("Insert char: ");
        scanf("%1s", &current);
        pushArray(current);
        checkArray();
    }
}

/**
 * Method that reset array to * values
 * @return void
 */
void resetArray()
{
    for (int i = 0; i < STRING_LENGTH; i++)
    {
        word[i] = '*';
    }
}
/**
 * Method that shift by 1 the array and append the value
 * @param char The value to append
 * @return void
 */
void pushArray(char value)
{
    for (int i = 0; i < STRING_LENGTH - 1; i++)
    {
        word[i] = word[i + 1];
    }
    word[STRING_LENGTH - 1] = value;
}
/**
 * Check if the array containts the STRING value
 * @return void
 */
void checkArray()
{
    if (strncmp(word, STRING, strlen(STRING)) == 0)
    {
        printf("Sequence found!\n");
    }
}

Implementation in Java

Operation

The program will ask you from time to time to insert a character, which will be inserted into an array which will in turn be examined to find the string requested. If the result is positive, the string “Sequence found!”

Code and explanation

Below is the source code

The TuringAutomaton class is the class where methods for automating the automaton are defined The main method contains the beginning of the program so it must be called to initialize the program is static and public

The constant variable STRING_TO_FIND contains the string to be checked in the input sequence

The pushInArray method is called to insert the character into the array according to the LIFO concept

The checkArray method is called to check the string in the array and return the outcome

import java.util.ArrayList;
import java.util.Scanner;

/**
 * Program based on the Turing Machine concept (console)
 * <br>
 * https://github.com/MarcoMihaiCondrache/TuringAutomaton
 *
 * @author  Condrache Marco Mihai
 * @version 1.0
 * @since   2019-11-28
 */

public class TuringAutomaton {
    /**
     * The final string that contains the string to find
     */
    private static final String STRING_TO_FIND = "aacbb";

    public static void main(String[] args) {
        String current;
        // full word to compare
        ArrayList<String> word = new ArrayList<String>();
        for (int i=0; i<STRING_TO_FIND.length(); i++){
            word.add("");
        }
        do {
            // insert and check the word with the new char
            Scanner reader = new Scanner(System.in);
            System.out.print("Insert char: ");
            current = reader.nextLine();
            word = pushInArray(word, current);
            if (checkWord(word)){
                System.out.println("Sequence found!\n");
            }
        }while (STRING_TO_FIND.contains(current));
        System.out.println(current);
    }

    /**
     * Method to check if the list contains
     * the STRING_TO_FIND and return true or false.
     *
     * @param  list The array to join to compare
     * @return  if the string contains the cost
     * @since 1.0
     */
    private static boolean checkWord(ArrayList<String> list){
        StringBuilder sb = new StringBuilder();
        for (String s : list)
        {
            sb.append(s);
        }
        return sb.toString().contains(STRING_TO_FIND);
    }

    /***
     * Method to push into array using LIFO concept
     *
     * @param list The array to push in
     * @param charT The char to insert
     * @return return the complete list
     */
    private static ArrayList<String> pushInArray(ArrayList<String> list, String charT){
        list.remove(0);
        list.add(charT);
        return list;
    }
}

Implementation in Python

Operation

The program will ask you from time to time to insert a character, which will be inserted into an array which will in turn be examined to find the string requested. If the result is positive, the string “Sequence found” will be printed

Code and explanation

Below is the source code

The constant variable STRING contains the string to be checked in the input sequence

The parameters and pushats in the array are requested each time with the relative function push_array (char) which shifts the array to the left by 1

The check_array () function checks the string inside the array and returns the outcome

"""
    * AUTHOR: Condrache Marco Mihai
    * CLASS: 3^A
    * DATE: 28/11/2019
    * DESCRIPTION: Program based on the Turing Machine concept (console)
    ? GITHUB: https://github.com/MarcoMihaiCondrache/TuringAutomaton
"""

# Define STRING variable to find inside characters
STRING = "aacbb"
# Define word variable that will contain the insert string
word = [""] * len(STRING)
# Define current variable that will contain the current char
current = STRING[0]


def push_array(char):
    """
    Push into array with Lifo concept

    :param char: The char to insert into the array
    :return:
    """
    word.pop(0)
    word.append(char)


def check_array():
    """
    Check if the array contains the sequence

    :return bool: Return the condition
    """
    return STRING in "".join(word)


while current in STRING:
    # Ask the char
    current = str(input("Insert char: "))
    # Insert it into the array
    push_array(current)
    # Check and eventually clear the array
    if check_array():
        print("Sequence found")

# If the while statement exits means that it has received an invalid char
print("Invalid char")

Implementation in Dart

Operation

The program will ask you from time to time to insert a character, which will be inserted into an array which will in turn be examined to find the string requested. If the result is positive, the string “Sequence found” will be printed

Code and Explanation

Below is the source code

import 'dart:io';

/// Constant string that have the value to find inside the array
/// Type: [String]
const STRING = "aacbb";

/// Main method that request the char every time
/// Return: [void]
void main() {
  /// Generate a list with empty space to prevent [Segmentation Fault]
  List<String> word = List.generate(STRING.length, (index) => "");

  /// First char of the string
  String current = STRING.split("")[0];
  do {
    print("Insert char: ");
    current = stdin.readLineSync();

    /// Save chars with the LIFO concept
    word.removeAt(0);
    word.add(current);

    /// Check the string
    if (checkArray(word)) {
      print("Sequence found!");
    }
  } while (STRING.contains(current));
}

/// Method that check if the array contains the [STRING]
/// Param: [List<String> array]
/// Return: [bool]
bool checkArray(List<String> array) {
  return array.join("") == STRING ? true : false;
}
